# Configuration Reference

Pincer uses [pydantic-settings](https://docs.pydantic.dev/latest/concepts/pydantic_settings/) for configuration. All settings are loaded from environment variables with the `PINCER_` prefix, or from a `.env` file.

## How Configuration Works

The `Settings` class in `src/pincer/config.py` defines all configuration with:

- **Type validation** via Pydantic (integers, floats, enums, paths, etc.)
- **Secret protection** using `SecretStr` — API keys are never accidentally logged
- **Default values** for every setting — only API keys are truly required
- **Singleton pattern** via `@lru_cache` — settings are loaded once and cached

```python
from pincer.config import get_settings

settings = get_settings()  # Cached singleton
```

## All Environment Variables

### LLM Provider

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_DEFAULT_PROVIDER` | `anthropic` \| `openai` | `anthropic` | Which LLM provider to use |
| `PINCER_ANTHROPIC_API_KEY` | SecretStr | `""` | Anthropic API key |
| `PINCER_OPENAI_API_KEY` | SecretStr | `""` | OpenAI API key (also used for Whisper voice transcription) |
| `PINCER_DEFAULT_MODEL` | str | `claude-sonnet-4-5-20250929` | Default model identifier |
| `PINCER_MAX_TOKENS` | int (1–128000) | `8192` | Max tokens per LLM response |
| `PINCER_TEMPERATURE` | float (0.0–2.0) | `0.7` | LLM temperature |

### Telegram

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_TELEGRAM_BOT_TOKEN` | SecretStr | `""` | Bot token from @BotFather |
| `PINCER_TELEGRAM_ALLOWED_USERS` | comma-separated ints | `[]` | User IDs allowed to use the bot (empty = allow all) |

### Agent Behavior

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_AGENT_NAME` | str | `Pincer` | Display name of the agent |
| `PINCER_MAX_TOOL_ITERATIONS` | int (1–50) | `10` | Max ReAct loop iterations before forcing a response |
| `PINCER_MAX_SESSION_MESSAGES` | int (10–500) | `50` | Max messages in session before trimming oldest |
| `PINCER_DAILY_BUDGET_USD` | float (≥0) | `5.0` | Daily spend limit in USD (0 = unlimited) |

### Search

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_TAVILY_API_KEY` | SecretStr | `""` | Tavily API key for rich web search (optional) |

### Shell

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_SHELL_ENABLED` | bool | `true` | Enable/disable shell execution tool |
| `PINCER_SHELL_TIMEOUT` | int (5–300) | `30` | Shell command timeout in seconds |
| `PINCER_SHELL_REQUIRE_APPROVAL` | bool | `true` | Require user approval before shell commands |

### Memory

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_MEMORY_ENABLED` | bool | `true` | Enable the memory system |
| `PINCER_SUMMARY_MODEL` | str | `claude-haiku-4-5-20251001` | Cheap model used for conversation summarization |
| `PINCER_SUMMARY_THRESHOLD` | int (≥5) | `20` | Summarize after this many messages |

### Storage & Logging

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `PINCER_DATA_DIR` | Path | `~/.pincer` | Data directory for database, logs, workspace |
| `PINCER_LOG_LEVEL` | `DEBUG` \| `INFO` \| `WARNING` \| `ERROR` | `INFO` | Logging level |

## Auto-Detection Logic

The configuration includes smart fallback behavior:

1. **Provider auto-switch**: If `PINCER_DEFAULT_PROVIDER=anthropic` but only `PINCER_OPENAI_API_KEY` is set, the provider automatically switches to `openai` (and vice versa).

2. **At least one key required**: If neither `PINCER_ANTHROPIC_API_KEY` nor `PINCER_OPENAI_API_KEY` is set, startup fails with a clear error.

3. **Directory auto-creation**: `data_dir`, `data_dir/logs`, and `data_dir/workspace` are created automatically on first run.

## Data Directory Layout

```
~/.pincer/
├── pincer.db          # SQLite database (sessions, costs, memories)
├── logs/              # Log files
└── workspace/         # Sandboxed file operations
    ├── uploads/       # Files uploaded by users
    └── exec_output/   # Files generated by python_exec tool
```
